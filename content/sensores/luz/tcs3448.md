# TCS3448 (ams-OSRAM)

Sucesor activo del [AS7343](as7343.md). Software-compatible con el AS7343 — los mismos drivers corren en ambos chips sin cambios.

Páginas del fabricante: [ams-OSRAM TCS3448](https://ams-osram.com/products/sensor-solutions/ambient-light-color-spectral-proximity-sensors/ams-tcs3448-spectral-color-sensor), [Datasheet (PDF)](https://look.ams-osram.com/m/1c24b057e65ee61e/original/TCS3448-14-Channel-multi-spectral-sensor.pdf)

## Specs

| Spec | Valor |
|---|---|
| Canales | 14 (11 espectrales 350–800 nm + IR + Clear + Flicker) |
| Voltaje del chip | 1.8V (requiere level shifter desde 3.3V de ESP32) |
| Interface | I2C @ 400kHz + I3C |
| Flicker detection | hasta 1 kHz (sampling a 2 kHz) |
| Package | OLGA 3.1×2.0×1.0 mm |

## Breakouts

No hay breakout conocido de Adafruit o SparkFun al momento. Requiere diseño propio de PCB con regulador 1.8V y level shifter (mismo requerimiento que AS7341/AS7343).

## Canales espectrales

Los canales son equivalentes a los del AS7343 (compatibilidad de software implica misma distribución espectral). Ver [AS7343 — canales](as7343.md) para la tabla detallada. Verificar con el datasheet si hay diferencias en longitudes de onda exactas.

## Diferencias con AS7343

| | AS7343 | TCS3448 |
|---|---|---|
| Estado | Discontinuado | **Activo** |
| Compatibilidad de software | referencia | ✅ drop-in |
| Flicker detection | ~100 Hz | **hasta 1 kHz** |
| Interface adicional | solo I2C | I2C + **I3C** |
| Breakout disponible | Adafruit #6477 | ninguno conocido |

## Diferencias con AS7341

Ver [AS7343 — diferencias con AS7341](as7343.md) — aplica igual al TCS3448, que hereda la misma arquitectura.

## Para PAR

Sin breakout disponible y sin calibraciones publicadas contra Apogee SQ-500. Para proyectos nuevos que requieran soldar su propio breakout, es preferible al AS7343 por ser el chip activo. Para proyectos que usan breakout off-the-shelf, el [AS7341 (#4698)](as7341.md) sigue siendo la opción documentada y validada.
